# 🧾 ระบบวางบิล & เก็บเงิน (Billing & Payment System)

Web Application สำหรับออกใบวางบิล/ใบแจ้งหนี้ รับชำระเงิน ออกใบเสร็จ ติดตามหนี้ และดูรายงานสรุปยอดขาย

## ✨ ฟีเจอร์ (ครบทั้ง 8 ข้อ)

| # | ฟีเจอร์ | รายละเอียด |
|---|---------|-----------|
| 1 | **สร้างใบวางบิลอัตโนมัติ** | สร้างเลขที่เอกสารอัตโนมัติ `INV-YYYYMM-####` คำนวณยอด/ภาษี/ส่วนลดให้ |
| 2 | **ใบเสร็จรับเงิน** | ออกใบเสร็จ `REC-YYYYMM-####` อัตโนมัติเมื่อรับชำระ พร้อมพิมพ์ได้ |
| 3 | **จัดการฐานข้อมูลลูกค้า** | เพิ่ม/แก้ไข/ลบ/ค้นหา ลูกค้า พร้อมยอดค้างชำระรายราย |
| 4 | **ตรวจสอบสถานะการชำระ** | ติดตามยอดชำระ/คงเหลือของแต่ละใบ รองรับชำระบางส่วน |
| 5 | **แจ้งเตือน & ติดตามเมื่อถึงกำหนด** | หน้า "แจ้งเตือน" รวมใบเกินกำหนด + ใกล้ครบกำหนด พร้อมส่งเตือนทั้งหมดได้ |
| 6 | **สถานะการชำระเงิน** | `รอชำระ` / `ชำระแล้ว` / `ค้างชำระ` (คำนวณอัตโนมัติจากกำหนดชำระ) |
| 7 | **ส่งเอกสารผ่านอีเมล / SMS** | ส่งใบวางบิล ใบเสร็จ หรือใบแจ้งเตือน ได้ทันที + บันทึก log การส่ง |
| 8 | **รายงาน & Dashboard** | สรุปยอดขาย รายรับ รายวัน/รายเดือน, กราฟ, ลูกค้ายอดสูงสุด, ส่งออก CSV |

## 🚀 วิธีรัน

```bash
npm install
npm run billing
```

เปิดเบราว์เซอร์: **http://localhost:4000**

- เข้าสู่ระบบเริ่มต้น: `admin` / `admin1234` (เปลี่ยนได้ในหน้าตั้งค่า)
- ข้อมูลถูกเก็บในไฟล์ SQLite `billing.db` (สร้างอัตโนมัติ ไม่ถูก commit)

## 📧 ตั้งค่าการส่งอีเมลจริง (ไม่บังคับ)

ถ้าไม่ตั้งค่า ระบบจะทำงานใน **โหมดจำลองการส่ง** (บันทึก log ไว้ดูได้ที่หน้าตั้งค่า)

ตั้งค่า environment variables เพื่อส่งอีเมลจริงผ่าน SMTP:

```bash
export SMTP_HOST=smtp.gmail.com
export SMTP_PORT=587
export SMTP_USER=you@gmail.com
export SMTP_PASS=your-app-password
export SMTP_FROM="ระบบวางบิล <you@gmail.com>"
npm run billing
```

> SMS ใช้กลไกเดียวกัน — ปัจจุบันบันทึก log/จำลองการส่ง สามารถต่อ API ผู้ให้บริการ SMS ได้ที่ฟังก์ชัน `sendNotification()` ใน `billing-server.js`

## 🗂️ โครงสร้างไฟล์

- `billing-server.js` — Backend (Express + better-sqlite3) REST API ทั้งหมด
- `billing.html` — Frontend (Single Page App, ภาษาไทย, รองรับ Light/Dark)
- `billing.db` — ฐานข้อมูล SQLite (สร้างอัตโนมัติ)
